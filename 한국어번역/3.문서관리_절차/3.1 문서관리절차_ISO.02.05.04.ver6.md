## 1. 목적

회사 내 유통 문서를 엄격하게 관리하고 편리하게 활용할 수 있도록 다음을 보장한다:

- 구버전 문서의 식별 및 오용 방지
- 업무 품질 관련 문서의 정확하고 완전하며 적시에 해당 부서에 배포
- 문서 발행 전 권한자의 승인 확보
- 핵심 업무 수행 장소에서 현행 문서의 즉시 활용 가능
- 오류 또는 구버전 문서의 즉각 제거로 혼용 방지

---

## 2. 적용 범위

회사 내에서 사용되는 내부 문서 및 외부 문서의 관리에 적용한다.  
회사 전 부서 및 팀에 적용된다.

## 5. 절차 내용

### 5.1 내부 문서 관리 (Kiểm soát tài liệu nội bộ)

#### 5.1.1 내부 문서 관리 흐름도

| 단계 | 담당 | 내용 | 관련 양식 |
|------|------|------|-----------|
| 1 | 해당 부서 | 문서 작성 요청 / 개정 요청 | - |
| 2 | 경영진 | 검토 (회의 중) | - |
| 3 | 경영진 | 승인 (회의 중) | - |
| 4 | 지정 담당자 | 문서 완성 | 완성 문서 |
| 5 | 경영진 | 최종 검토 및 승인 | 완성 문서 |
| 6 | ISO 간사 / 지정 담당자 | 관리 인감 날인 후 사본 배포 | 관리 인감, 문서 배포 전표 |
| 7 | ISO 간사 / 지정 담당자 | 내부 문서 목록 갱신 | 문서 목록 |
| 8 | ISO 간사 / 지정 담당자 | 기록 보관 | 관련 기록 일체 |

#### 5.1.2 외부 문서 관리 (Kiểm soát tài liệu bên ngoài)

- 외부 문서의 관리 인감 날인은 본 절차서 4.2항 기준을 따른다.
- 외부 문서를 수령하여 사용하는 부서는 **외부 문서 목록**을 작성하여 관리한다.

### 5.2 문서 표시 형식

- **절차서**: 본 문서의 1·2페이지 형식과 동일하게 작성
- **양식**: 3가지 유형
  - 정부 기관 발송 공문: 공문 형식 기준 준수
  - 거래처 발송 문서: 회사 로고, 명칭, 주소, 전화번호 포함
  - 내부 업무용 문서: 각 절차서에서 규정한 양식 사용, 로고·코드·발행일 인쇄

### 5.3 문서코드 규정

**기본 형식**: `A.XX.YY.ZZ`

| 기호 | 의미 |
|------|------|
| A | 부서코드 (위 약어표 참조) |
| XX | 문서 계층 코드 |
| YY.ZZ | 문서 일련번호 |

**계층 코드:**

| 코드 | 문서 유형 |
|------|-----------|
| 01 | 품질방침, 환경방침, 환경목표 |
| 02 | 절차서 |
| 03 | 규정, 규칙, 지침서, 설명서 |
| 04 | 기록, 양식 |

> **예시**: `GA.01.01` = 총무·인사팀 품질방침

**복수 제품 적용 시**: `A.XX.YY.ZZ/P` (P = 제품 식별 코드)  
> **예시**: `QM.03.00.06/NX4-P01` = NX4 모델 PQC 공정 제품 검사 기준서

### 5.4 문서 구성 내용

#### 5.4.1 절차서 구성 내용

| 항목 번호 | 항목명 | 내용 |
|-----------|--------|------|
| (1) | 목적 | 문서의 목적 기술 |
| (2) | 적용 범위 | 문서 적용 범위 기술 |
| (3) | 참조 문서 | 작성·실행 시 참조한 문서 목록 |
| (4) | 정의 | 신규 개념 또는 약어 (해당 시) |
| (5) | 내용 | 절차의 세부 내용 |
| (6) | 기록 관리 | 보관 기록, 장소, 보존 기간 |
| (7) | 부속서 | 첨부 양식 및 설명 문서 (해당 시) |

#### 5.4.2 규정·지침서 구성 내용

규정 및 지침서는 절차서와 동일하거나 그보다 적은 항목으로 구성 가능

### 5.5 문서 작성 및 승인 책임

| 문서 유형 | 작성 | 검토 | 승인 |
|-----------|------|------|------|
| 규정, 절차서, 지침서, 직무기술서 | 지정 담당자 | ISO 추진팀 | 고위 관리자 |
| 일반 기록 양식 | 지정 담당자 | ISO 추진팀 | 부서장 또는 팀장 |

### 5.6 문서 배포

- 승인 완료 후 ISO 간사가 관리 인감을 날인한 사본을 관련 부서에 배포
- 회사 공용 드라이브에 배포하며, 각 부서별 보관·유지 담당자 1명 지정
- 각 폴더에는 내부 또는 외부 문서 목록을 작성하여 보관
- 외부 제공은 대표이사 승인 필요

### 5.7 문서 개정 및 갱신

- 사용 중 개정이 필요할 경우 최초 발행 시와 동일한 검토·승인 절차 적용
- 개정 내용은 각 문서 1페이지 개정 이력표에 기록
- 개정된 문서는 내부 문서 목록에 갱신
- 참조 목적으로 보관하는 구버전 문서에는 **"구버전 문서"** 표시
